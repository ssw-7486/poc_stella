## Overview

Step 5 allows customers to estimate processing volume per Line of Business for capacity planning. This wireframe shows both collapsed (panel hidden) and expanded (panel visible) states of the collapsible side panel pattern.

**CRITICAL:** Number of LOB sections is dynamically generated based on Step 1 field "No. of Lines of Business Being Processed". If Step 1 = "3", Step 5 shows 3 LOB sections. If Step 1 = "1", Step 5 shows 1 LOB section.

## Behavioral Notes

### Panel Toggle

**Default State:** Panel hidden, [Show â†’] button visible
**LocalStorage Persistence:** `wizard_step5_panel_visible` (boolean)
**Layout:**
- Panel hidden: 100% width for main content
- Panel visible: 70% main content, 30% side panel

### Dynamic LOB Sections

**CRITICAL VALIDATION:** Number of LOB sections must match Step 1 field "No. of Lines of Business Being Processed"

**Examples:**
- Step 1 LOB count = 1 â†’ Step 5 shows 1 LOB section
- Step 1 LOB count = 3 â†’ Step 5 shows 3 LOB sections (as shown in wireframe)
- Step 1 LOB count = 10 â†’ Step 5 shows 10 LOB sections (scrollable)

**Implementation Note:** Must read Step 1 data from wizard state to determine section count

### Real-Time Volume Calculation

**As User Types:** Total estimated volume updates automatically
**Calculation:** Sum of all "Expected Monthly Volume" fields
**Display:** "Total Estimated Volume: X documents/month"

**Side Panel Update:** Side panel summary also updates in real-time when panel is visible

### Skip This Step

**[Skip This Step] Button:**
- Shows warning dialog: "Volume estimates help us plan capacity. Are you sure you want to skip?"
- Options: "Go Back" or "Skip Anyway"
- If user skips, Step5Data is saved with `skipVolumeEstimate: true`
- All volume fields left empty

**Validation:**
- If user enters any volume data, [Skip This Step] button is disabled
- If user clicks [Next] with partial data, show validation errors
- Either skip entirely OR fill all required fields (LOB Name + Volume for each section)

### Field Validation

**LOB Name:**
- Required if proceeding (not if skipping)
- Minimum 2 characters
- Duplicate LOB names allowed (same LOB in multiple sections is valid use case)

**Expected Monthly Volume:**
- Required if proceeding (not if skipping)
- Must be positive integer
- Minimum: 1
- Maximum: 10,000,000
- Accepts commas (e.g., "150,000" or "150000" both valid)

**Peak Processing Period:**
- Optional (can be left blank)
- Free text field
- Examples: "End of month", "Q4", "Holiday season", "Monday mornings"

**Gotcha:** If Step 1 data is missing or invalid, Step 5 cannot render - show error and redirect to Step 1
