## Overview

Step 7 allows customers to review all previous entries, accept security/privacy policies, and provide electronic signature before finalizing workflow setup. This wireframe shows both collapsed (panel hidden) and expanded (panel visible) states of the collapsible side panel pattern.

**KEY DESIGN DECISION:** Option 3 selected - Single column with card-style summaries
**3 Options Evaluated:**
- Option 1: 2-column grid for step summaries (rejected)
- Option 2: Panel visible with compact side panel summary (rejected)
- **Option 3: Single column with card-style summaries (APPROVED)**

## Behavioral Notes

### Panel Toggle

**Default State:** Panel hidden, [Show â†’] button visible
**LocalStorage Persistence:** `wizard_step7_panel_visible` (boolean)
**Layout:**
- Panel hidden: 100% width for main content
- Panel visible: 70% main content, 30% side panel

### Step Summary Cards

**6 Cards Total:** One per step 1-6
**Card Structure:**
- Step number and title (e.g., "1. COMPANY INFO")
- [Edit] button to return to that step
- Compact bullet-point summary using summary text from each step

**[Edit] Button Behavior:**
- On Click: Navigate to that step (e.g., click [Edit] on Step 3 â†’ go to Step 3)
- Preserve all data
- User can modify configuration
- On Next from that step: Save changes and return to Step 7
- Step 7 summary updates with new values

**Step 2 Edit Warning:**
- If user clicks [Edit] on Step 2 (Template Selection) and changes template
- Show confirmation dialog: "Changing your template will reset your settings in the following steps. Continue?"
- Options: "Continue" (reset Steps 3-6) or "Go Back" (keep current)

### Security & Privacy Policies

**4 Policy Cards:**
1. Data Processing Agreement (DPA)
2. Service Level Agreement (SLA)
3. Compliance Requirements
4. Audit & Retention Policy

**Card Structure:**
- Checkbox (must be checked)
- Policy name (title)
- Detailed description (2-3 sentences)
- [View Document â†’] link

**[View Document â†’] Link:**
- Opens policy document in new tab
- URL format: `/policies/dpa?country=[country_code]` (country-specific content based on Step 1)

**Country-Specific Content:**
- Policy descriptions dynamically loaded based on Step 1 Country field
- Example: GDPR mentioned for EU countries, CCPA for California
- Implementation must load correct policy content per country

### Electronic Signature

**Field:** "Accepted By (Full Name)" with asterisk (required)
**Validation:**
- Minimum 2 characters
- Cannot be empty
**Helper Text:** "Your electronic signature confirms acceptance of all policies"

### Complete Setup Button

**Enabled State:**
- All 4 policy checkboxes checked
- "Accepted By" field filled (â‰¥2 characters)

**Disabled State:**
- Button grayed out
- Hover tooltip: "Please accept all policies and provide your name"

**On Click (when enabled):**
1. Show loading state (1-2 seconds): "Creating your workflow..."
2. Save Step7Data with timestamp
3. Create workflow with all step 1-7 data
4. Initialize audit trail
5. Show success toast: "Workflow setup complete!"
6. Redirect to Dashboard (`/dashboard`)

**On Dashboard:**
- New workflow appears in "Active Workflows" section
- Show welcome message or tutorial prompt (optional)

### Footer Actions

**[Cancel]:**
- Shows confirmation dialog: "Cancel setup? All progress will be lost."
- Options: "Continue Setup" or "Cancel Setup"
- If confirmed: Return to Dashboard, discard all progress

**[Save & Exit]:**
- **Disabled on Step 7** (workflow must be completed or cancelled, cannot save incomplete)
- Button is hidden or grayed out

**[â† Back]:**
- Returns to Step 6
- Preserves all Step 7 data (checkboxes, signature)

## Key Learnings & Gotchas

### 1. Option 3 Selected After Evaluating 3 Options

**Options Evaluated:**
- **Option 1:** 2-column grid for step summaries - Rejected (cramped)
- **Option 2:** Panel visible with compact side panel summary - Rejected (not enough detail)
- **Option 3:** Single column with card-style summaries - **APPROVED** (best readability)

**Rationale:** Single column provides best readability for comprehensive summaries. Policy cards need full width for detailed descriptions.

### 2. Step 2 Edit Warning Required

**Critical:** If user changes template in Step 2, Steps 3-6 configuration may become invalid
**Must Show Dialog:** "Changing your template will reset your settings in the following steps. Continue?"
**Gotcha:** Implement template change detection logic

### 3. Country-Specific Policy Content

**Dynamic Content:** Policy descriptions must be loaded based on Step 1 Country field
**Examples:**
- EU countries: Mention GDPR
- California: Mention CCPA
- Healthcare sector: Emphasize HIPAA
**Gotcha:** Must have policy content variants per country/region

### 4. Save & Exit Disabled on Step 7

**Reason:** Workflow must be either completed or cancelled, cannot save partial workflow
**UI:** Hide or gray out [Save & Exit] button on Step 7
**Gotcha:** Don't forget to disable this button (it's enabled on all other steps)

### 5. Summary Text Must Reflect Actual User Inputs

**Dynamic Summaries:** All 6 step summary cards must show actual user data from wizard state
**Examples:**
- Step 1: Show actual company name, country, LOB count
- Step 3: Show actual template count and average accuracy
- Step 5: Show actual LOB names and volumes
**Gotcha:** Must read data from wizard state, not hardcoded values

### 6. Complete Setup Button Validation

**Two Conditions:** All 4 checkboxes + name field
**Real-Time Validation:** Button enable/disable updates as user types or checks boxes
**Gotcha:** Must implement real-time validation logic, not just on-click validation
